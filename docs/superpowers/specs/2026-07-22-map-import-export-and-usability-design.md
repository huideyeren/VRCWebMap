# 地図の初回登録体験、KML 相互運用、背景地図の設計

**作成日:** 2026-07-22
**状態:** ユーザー合意済み・実装計画待ち

## 目的

地点を登録する最初の操作を迷いにくくし、既存の Google My Maps と uMap の地図資産を安全に持ち込めるようにする。現行の Leaflet による軽い地図体験は維持し、Google Maps API や OpenStreetMap 本体への自動投稿には依存しない。

## 利用者体験

### 背景地図とズーム

- 背景地図は「標準地図」と「航空写真」を切り替えられる。
- 標準地図は現在の OpenStreetMap タイルを使う。航空写真は国土地理院の全国最新写真を使い、必要な出典表示を常に表示する。
- 日本周辺を対象にした表示範囲、最小ズーム、最大ズームを明示的に設定し、地球規模まで縮小したり、画像タイルを意味なく拡大したりしない。
- ズームは 0.25 段階にする。右側に視認しやすい `+` と `-` の操作、現在の縮尺、到達した上限・下限を表示する。キーボード、ホイール、トラックパッド、ピンチ操作も同じ段階に揃える。
- 右クリック登録は維持し、登録フォームから現在の地図中心を座標へ反映できる操作も追加する。

### KML/KMZ インポート

- 既存の `KmlSpotImportParser` を共通パーサーとして維持する。対応形式、5 MiB 上限、5,000 Point Placemark 上限、WGS84 座標、非対応要素の警告も維持する。
- 管理画面専用の KML インポートを廃止し、通常の地図画面に共通の「インポート」導線を置く。
- Discord ログイン済みかつ VRChat Display Name を登録済みの利用者は、管理者を含めて同じインポート体験を利用できる。未ログインまたは表示名未登録の利用者は既存の書き込み制約どおり実行できない。
- プレビューは各候補と既存 Spot の最短距離を測り、50 m 以下の候補を「重複の可能性あり」と表示する。名称、説明、地域コードは重複判定に使わない。
- 重複の可能性がある候補は初期状態で選択されない。利用者は候補に対応する既存 Spot の名称、距離、位置を確認したうえで、個別に選択してインポートできる。
- 50 m より遠い候補は初期状態で選択する。利用者は任意に選択解除できる。
- 実行時には KML を再解析し、最新の既存 Spot に対して同じ距離照合を行う。利用者の確認は候補番号だけでなく、確認時点の近接 Spot ID の集合として受け取る。プレビュー時には存在しなかった 50 m 以下の Spot が生じた場合、実行を止めてプレビューを更新し、利用者の再確認を求める。これによりプレビューと実行の間の二重登録を防ぐ。
- 距離は地表面距離として計算し、境界値 50.0 m を重複候補に含める。高速化が必要になった場合は、距離計算前に緯度経度グリッドで候補を絞り込むが、最終判定は距離計算で行う。

### KML エクスポート

- 地図または Spot 一覧から、利用者が選択した公開 Spot を KML としてダウンロードできる。所有者や管理者だけに限定しない。
- 出力は Google My Maps と uMap の両方で使える Point Placemark の KML にする。Spot 名は `name`、説明は `description`、座標は WGS84 の `longitude,latitude,0` に対応付ける。
- 各 Placemark の `description` には、Spot 本文に加えて「出典: VRC Web Map」「登録者: {VRChat Display Name}」「元Spot: {公開URL}?spotId={Spot ID}」を必ず記載する。これにより、外部地図へ取り込んだ後もデータの由来、登録者、原典へ遡れる。
- 元Spotへのリンクはサーバー設定の `App:PublicBaseUrl` から生成する。リバースプロキシ、開発環境、外部地図のいずれでも正しい絶対URLを出せるよう、HTTP request の Host ヘッダーやブラウザーから受け取った URL は出典リンクに使わない。
- 各 Placemark に VRC Web Map の Spot ID を拡張データとして含める。初期版の重複判定は距離のみを使うため、この ID は将来の往復同期や利用者の照合のために保持する。
- uMap の正式な相互運用形式は KML とする。GeoJSON は uMap 利用者への利便性を高める後続拡張として扱い、この変更には含めない。
- OpenStreetMap 本体への投稿、OSM API 認証、OSM XML の投稿用データ生成はこの変更の対象外とする。uMap は OSM を背景地図に使う独立したカスタム地図サービスとして扱う。

## アーキテクチャー

### Contracts と UseCases

- KML の候補と選択状態は `Contracts/Spots/` に置き、HTTP 固有の型を持ち込まない。
- `PreviewKmlImport` は、候補ごとの安定した入力順 ID、選択初期値、近接する既存 Spot の要約を返す。
- `ImportKmlSpots` は、ファイル、既定地域、利用者が明示的に選択した候補 ID、プレビューを識別する値を受け取る。UseCase は実行時の再照合で新しい近接候補を見つけたら、再確認が必要な業務エラーとして返す。
- 距離による重複候補判定は `UseCases/Spots/` の小さな純粋責務として切り出す。KML XML の解析、認可、永続化、UI 表示から独立させる。
- KML エクスポートは `Contracts/Spots/ExportKmlSpots.cs` と `UseCases/Spots/ExportKmlSpotsUseCase.cs` に置く。選択された公開 Spot を読み取り、KML 文字列またはダウンロード用の内容を返す。
- `ExportKmlSpotsUseCase` は公開 URL 設定と登録者情報を明示的に依存として受け取り、出典・登録者・元Spot URL を KML の説明へ組み立てる。URL の基点は `Options/AppOptions.cs` の設定境界から供給する。
- `Endpoints/Web/SpotsEndpoints.cs` は各 UseCase を公開するだけにし、距離判定やKML組み立てを持たない。

### フロントエンド

- 地図画面にインポート用パネルと、Spot 選択型のエクスポート操作を置く。管理画面の KML タブは削除し、管理者にも同じ共通パネルを使わせる。
- インポートのプレビューでは、通常候補と重複候補を別表示にし、重複候補は未選択状態で表示する。候補の地図上の位置と既存 Spot を比較できるようにする。
- Leaflet の背景レイヤは定義をまとめ、標準地図と国土地理院航空写真の切替状態を画面内で表示する。各レイヤの attribution は Leaflet の attribution control から隠さない。

## エラーと結果表示

- ファイル形式、サイズ、座標、地域コード、対応していない Placemark の既存バリデーションは維持する。
- インポート結果は、作成件数、利用者が選択せずスキップした件数、非対応件数、再確認が必要になった候補、警告を区別して表示する。
- 新しい近接候補により実行を中止した場合、データは一件も保存しない。更新済みのプレビューだけを表示し、利用者が改めて選択してから実行する。
- KML エクスポートで存在しない Spot ID が選ばれた場合は、その ID を結果に含めず警告する。少なくとも1件が有効なら有効な Spot を出力する。

## テスト

- `KmlSpotImportParser` の既存テストを維持し、既存 KML/KMZ 互換性を確認する。
- 距離候補判定は、50 m 未満、ちょうど 50 m、50 m 超、同座標、複数の既存 Spot のうち最も近いものを UseCase 単位でテストする。
- プレビューでは重複候補が未選択になること、利用者が明示選択した候補だけが作成されることをテストする。
- プレビュー後に新しい近接 Spot が追加された場合、保存せず再確認を返すことをテストする。
- KML エクスポートは、選択 Spot の名称、説明、座標、拡張 Spot ID、出典、登録者名、設定済み公開URLによる元Spotリンクが正しい KML になること、選択されなかった Spot を含めないことをテストする。
- フロントエンドは、レイヤ切替、ズームの境界、重複候補の既定未選択、選択エクスポートの入力状態をテストする。

## 非対象

- Google Maps API の埋め込みや Google の衛星タイル直接利用
- OpenStreetMap 本体への自動投稿、OSM API 認証、投稿用 OSM XML 生成
- uMap とのアカウント連携、サーバー間同期、GeoJSON エクスポート
- KML の LineString、Polygon、MultiGeometry、NetworkLink、外部参照の取り込み
